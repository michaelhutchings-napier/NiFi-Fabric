NiFi autoscaling Kubernetes

Scale Apache NiFi on Kubernetes with a controller-owned safety model.

NiFi-Fabric treats autoscaling as lifecycle work, not just replica math. The controller can recommend or enforce actions while keeping scale-down conservative and one node at a time.

Autoscaling model

Why scaling NiFi needs more than a generic HPA

Apache NiFi is a stateful system, so autoscaling needs to respect rollout ordering, restart safety, and competing lifecycle work. NiFi-Fabric keeps the controller as the only component that makes real scale changes in managed mode, which makes the behavior easier to explain and safer to operate.

  • Disabled keeps replica counts fixed.
  • Advisory records recommendations for operator review.
  • Enforced allows controller-owned runtime actions.
  • Scale-down remains conservative and one node at a time.

KEDA can still participate by writing scaling intent into the managed API surface, but it does not take direct ownership of the NiFi StatefulSet. That preserves one clear source of runtime authority.
