NiFi 2 on AKS

Deploy Apache NiFi 2 on AKS with the primary supported platform shape.

AKS is the primary supported target environment for NiFi-Fabric. The recommended path combines the platform chart, cert-manager-first TLS, and PVC-backed NiFi repositories.

AKS deployment shape

What to prepare before the first install

For AKS, start with a cluster that can provision the PVCs NiFi needs, make sure the kubelet identity can pull both the controller and NiFi images, and install cert-manager before the first Helm release. That keeps the deployment path aligned with the core product docs and support position.

  • Use charts/nifi-platform in managed mode.
  • Install cert-manager and create or choose an Issuer first.
  • Use internal ClusterIP service exposure as the default starting point.
  • Add ingress or a load balancer only when the deployment needs it.

This path works well for GitOps because the chart values, TLS references, and exposure changes stay declarative and easy to review in source control.

AKS checklist

  • A PVC-capable StorageClass for NiFi repositories.
  • Image access for both controller and NiFi containers.
  • cert-manager installed and ready.
  • Issuer or ClusterIssuer selected for workload certificates.

Suggested AKS install command

The standard quickstart profile is the recommended starting point for evaluation and early platform bring-up on AKS. 

helm upgrade --install nifi charts/nifi-platform \
  --namespace nifi \
  --create-namespace \
  --set global.nifiFabric.installProfile=quickstart-cert-manager \
  --set nifi.tls.certManager.issuerRef.name=nifi-ca
