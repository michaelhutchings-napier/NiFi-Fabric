Apache NiFi 2 on Kubernetes

Run Apache NiFi 2 on Kubernetes without a heavyweight operator stack.

NiFi-Fabric is a Helm-first Apache NiFi 2 Kubernetes platform with cert-manager-first TLS, GitOps-friendly workflows, and a thin operator focused on rollout and lifecycle safety.

Why teams choose NiFi-Fabric

NiFi-Fabric is built for teams that want Apache NiFi 2 on Kubernetes without taking on a large CRD-heavy operator model. The product stays Helm-first, keeps security and restart safety front and center, and adds the small amount of controller logic needed for production lifecycle work.

Built for real platform work

  • Helm-first installs and upgrades for Apache NiFi 2 on Kubernetes
  • cert-manager-first TLS with restart-safe certificate rotation
  • Thin operator logic for rollout, restore, and lifecycle safety
  • Native NiFi 2 metrics with ServiceMonitor-friendly observability
  • Managed auth, autoscaling, and optional trust-manager integration
  • AKS-first, OpenShift-friendly, with optional service mesh profiles

Core features

Helm-first install

Standard Kubernetes resources stay in Helm, so the default path is explicit, GitOps-friendly, and easy for platform teams to operate.

TLS and trust distribution

TLS is enabled by default with cert-manager-first certificate ownership, restart-safe rotation handling, and optional trust-manager integration for shared CA distribution.

Native NiFi 2 metrics

Prometheus scraping works against the secured NiFi 2 API with ServiceMonitor support and no reverse-proxy sidecar requirement.

Safe autoscaling

Advisory and enforced autoscaling modes help teams scale up quickly while keeping scale-down behavior safety-aware.

Access and mesh profiles

Start simple, then move to managed OIDC or LDAP installs with clear ownership boundaries for auth and TLS inputs, plus optional Linkerd and Istio workload profiles when your cluster already runs a supported service mesh.

Lifecycle safety

The controller stays thin and focuses on the work Helm cannot do safely on its own: rollout sequencing, restart safety, and recovery-aware behavior.

Quick install

Start with the supported quick evaluation path and move to explicit values files, OIDC, or LDAP when you are ready for production rollout planning. 

helm upgrade --install nifi charts/nifi-platform \
  --namespace nifi \
  --create-namespace \
  --set global.nifiFabric.installProfile=quickstart-cert-manager \
  --set nifi.tls.certManager.issuerRef.name=nifi-ca
